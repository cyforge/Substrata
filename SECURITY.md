# Security Policy

Substrata is a payment-oriented SDK. Please report security issues privately.

## Reporting a Vulnerability

- Do **not** open a public issue for suspected vulnerabilities.
- Prefer GitHub private vulnerability reporting for this repository if it is enabled.
- If private reporting is not available, contact the repository owner or maintainer through the private contact method configured on the repository host before sharing details publicly.

## What To Include

Please include:

- a clear description of the issue
- the affected files, commands, or flows
- steps to reproduce when possible
- the impact you believe the issue has
- any mitigations or patches you already tested

## Triage and Response

- Project maintainers triage incoming reports.
- We will acknowledge valid reports as quickly as practical.
- We will coordinate disclosure timing with the reporter when a fix is needed.

## Supported Versions

This project is pre-1.0 and does not yet provide long-term support guarantees.
Security fixes will be made on the active default branch unless maintainers state otherwise.

## Scope Notes

- Security reports should focus on vulnerabilities, not general bugs or feature requests.
- For non-security defects or questions, use the normal repository issue flow.
